Transforming Dependent Variable for Skewness and Applying Regression Model¶

As it can be seen from the skewness value and boxplot, the dependent variable needs to be transformed in some manner to bring it to a Normally Distributed Form to apply a large number of tests which require Normality. To do this, we can use Box Cox Transformation to find the lambda value for us to decide what transformation needs to be performed on the dependent variable.

Box Cox Transformation works only for the positive values and not for negative values. So, for this data Box Cox Transformation won't work. Instead of it, we can use Yeo-Johnson Transformation method using PowerTransformer from the sklearn preprocessing module to work with negative values.

Power Transform will transform the Dependent data automatically instead of finding the lambda value and manually transforming it.

Approach for the Analysis¶

To start with I looked at the data which was only 2 variables 1 Independent and 1 Dependent variable. Loading the data to understand it better before jumping to fit a trend line was important.

The very 1st steps I follow during the data exploration is to ensure there are no null values, look for any outliers in the data using boxplots or z-score approach or use domain knowledge to make the data clean of outliers. For this data, I went ahead with creating histograms, histograms to visually see the distribution of data along with calculating the skewness for checking Normality. After that I checked for relationship between the X and the Y variables using scatter plot along with correlation coefficient value.

Once the exploratory analysis is done, I knew I had to fit a trend line which meant building a Regression model for predicting Y using X. As the data was small, I tried to fit a linear trend line along with polynomials of the order of 2, 3, 4 to see what best line or curve could be fitted. I discovered that fitting the polynomial hardly increased the R2 by 1% at the max from the linear line, so went ahead with the simplest solution of Linear Trend line to fit the data.

Statmodels library regression output gives a lot more information other than R2 and coeffcients values so decided to go with it initially and later on compare it with the results obtained from Sklearn library regression model.

Once the trend line was fitted and the slope, intercept values came out to be same for the libraries models, I went ahead with addressing a problem I saw earlier with Dependent Variable being Right Skewed.

To solve the skewed Y variable problem, I initially tried Box Cox Transformation to find the lambda value for transforming Y variable. But for Box Cox Transformation all the values needed to be positive for Y variable, so instead I used Power Transformer from Sklearn preprocessing module to transform the Y variable automatically using the Yeo-Johnson method which works for both positive and negative values.

Build a Statsmodels Regression model on Transformed Y and tested it on R2, RMSE for train and test data which performed almost the same as previous model.

Finally, checked if the assumptions of linear regression are satisfied by both the models using Normality Check for the Residuals and Q-Q plot which proved that the assumptions are not violated and the trend line build is good.

In general, for any problem I always try to follow CRISP-DM approach by first understanding the data throughly, most important step being Data Cleaning involving imputing missing values, outlier treatment, exploring the data for hidden patterns using EDA, feature engineering to create new features to enahnce model performance, preprocessing the data as required by the ML algorithms including one hot encoding, label encoding, scaling to build ML algorithms starting first with simpler models such as Logistic/Linear Regression, Naive Bayes and moving ahead with Random Forests, Gradient Boosting later on the in the process. Training these models, hyper tuning hyperparamters, evaluating the model's performance on test data using the appropriate metric such as Recall, Precision, Accuracy to make generalized predictions. Based on the problem to be solved like regression or classification, size of the data and other factors different ML algorithms could be choosen and fine tuned to give generalized prediction.

Interesting Findings and Final Conclusion¶

  1. Interesting findings that came up were that R2 was only 52% which is almost like random guess output.
  1. After fitting the trend line using Statsmodels library, model performed equally well on the train and the test data giving almost same R2 and RMSE.
  1. Statsmodels and Sklearn library Regression model gave the same predictions for slope and intercept along with R2 and RMSE with Statsmodels giving other important information.
  1. Treating the Right skewness of the Y variable did not have a drastic change in the model's performance in terms of R2 and RMSE. One thing to note is that interpretation of the coefficients could not be done for the transformed Y variable model making it less intutive.
